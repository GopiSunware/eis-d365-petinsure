Role-Based Access Control (RBAC)

Administrator

  • Full CRUD on all entities
  • AI provider configuration
  • Rating factor updates
  • User & role management
  • View all audit logs
  • Cost monitoring

Approver

  • Approve config changes
  • Review AI model changes
  • Approve rating updates
  • View approval history
  • Cannot create changes
  • Cannot manage users

Config Manager

  • Create config changes
  • Update AI settings
  • Modify claims rules
  • View audit logs
  • Cannot approve own changes
  • Cannot manage users

Viewer

  • Read-only configuration
  • View cost reports
  • View audit logs
  • View pipeline status
  • Cannot modify anything
  • Cannot trigger pipelines

Maker-Checker Approval Workflow

Configuration changes to ai_config, claims_rules, rating_config, and policy_config require approval.

1
Config Manager Submits Change

Creates a pending configuration change request (maker)

2
System Creates Approval Request

Change stored as "pending" with full audit trail

3
Approver Notified

Notification sent to approver role for review

4
Review & Decision

Approver reviews and approves or rejects (checker)

5
Configuration Applied

If approved, config applied and audit log updated
